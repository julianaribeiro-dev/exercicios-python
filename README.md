# exercicios-python
Repositório com exercícios propostos em sala de aula para apresentação de códigos em Python. 

EXERCÍCIOS

1. Faça um programa que converta metros para centímetros.

2. Faça um Programa que peça o raio de um círculo, calcule e mostre sua área.

3. Faça um Programa que calcule a área de um quadrado, em seguida mostre o dobro desta área para o usuário.

4. Faça um Programa que pergunte quanto você ganha por hora e o número de horas trabalhadas no mês. Calcule e mostre o total do seu salário no referido mês.

5. Faça um Programa que peça a temperatura em graus Fahrenheit, transforme e mostre a temperatura em graus Celsius. 
FÓRMULA: C = 5 * ((F-32) / 9).

6. Faça um Programa que peça a temperatura em graus Celsius, transforme e mostre em graus Fahrenheit.

7. Faça um Programa que peça 2 números inteiros e um número real. Calcule e mostre:
   A) o produto do dobro do primeiro com metade do segundo .
   B) a soma do triplo do primeiro com o terceiro.
   C) o terceiro elevado ao cubo.

9. Tendo como dados de entrada a altura de uma pessoa, construa um algoritmo que calcule seu IMC (índice de massa corporal).
FÓRMULA: peso/ (altura **2)

10. Tendo como dado de entrada a altura (h) de uma pessoa, construa um algoritmo que calcule seu peso ideal, utilizando as seguintes fórmulas:
  A) Para homens: (72.7*h) - 58
  B) Para mulheres: (62.1*h) - 44.7

10. Um sistema de cadastro precisa classificar automaticamente os usuários com base na idade informada. O usuário deverá informar sua idade em anos inteiros, e o sistema deve classificá-lo conforme as seguintes regras:
  De 0 a 12 anos: Criança
  De 13 a 17 anos: Adolescente
  De 18 a 59 anos: Adulto
  60 anos ou mais: Idoso

  O programa deve exibir a categoria correspondente à idade informada.
  
11. Um sistema de saúde deve calcular o Índice de Massa Corporal (IMC) de um usuário. O usuário deve informar seu peso (em kg) e altura (em metros). O IMC é calculado pela fórmula: IMC = peso / (altura²).

  Após o cálculo, o sistema deve classificar conforme as regras:
  IMC menor que 18,5: Abaixo do peso
  IMC entre 18,5 e 24,9: Peso normal
  IMC entre 25,0 e 29,9: Sobrepeso
  IMC igual ou maior que 30,0: Obesidade
  O programa deve exibir o valor do IMC e sua classificação.
  
12. Um sistema possui uma senha cadastrada previamente com o valor "1234". O usuário deve informar uma senha para acessar o sistema. As regras são:
  Se a senha informada for exatamente igual a "1234", o acesso deve ser permitido
  Se a senha for diferente, o acesso deve ser negado
  Se o usuário não digitar nenhum valor (entrada vazia), a senha deve ser considerada inválida
  O programa deve exibir a mensagem correspondente ao resultado da verificação.

13.  Um banco cobra tarifas mensais diferentes de acordo com o tipo de conta do cliente e seu saldo médio mensal. O sistema deve ler o tipo de conta (1 para conta comum, 2 para conta premium) e o saldo médio, aplicando as seguintes regras:

Para conta comum:
  Saldo médio menor que R$1.000,00: tarifa de R$25,00
  Saldo médio entre R$1.000,00 e R$5.000,00: tarifa de R$15,00
  Saldo médio acima de R$5.000,00: isento
Para conta premium:
  Saldo médio menor que R$5.000,00: tarifa de R$20,00
  Saldo médio igual ou maior que R$5.000,00: isento
  O programa deve exibir o valor da tarifa a ser cobrada.

14. Uma plataforma de vídeos curtos precisa calcular o valor que um criador de conteúdo irá receber com base no desempenho de um vídeo. O sistema deve ler a quantidade de visualizações e aplicar as seguintes regras de monetização:
  Até 10.000 visualizações: não há monetização
  De 10.001 até 100.000 visualizações: recebe R$0,02 por visualização
  De 100.001 até 1.000.000 visualizações: recebe R$0,03 por visualização
  Acima de 1.000.000 visualizações: recebe R$0,05 por visualização
  Além disso, o sistema deve aplicar um bônus: Se o vídeo tiver mais de 500.000 visualizações, adicionar um bônus fixo de R$500,00
  O programa deve calcular e exibir o valor total que o criador irá receber.
